Ingredients

  • 1 TypeScript back-end codebase
  • Semicolons to taste

What is RPC?

“[...] a remote procedure call is when a computer program causes a procedure [...] to execute [...] on another computer [...] without the programmer explicitly coding the details for the remote interaction.”
— Wikipedia, 2011

What is tRPC?

“tRPC is a TypeScript library that allows programmers to define routes on a web server and call them on a client, while providing type safety on both ends automatically.”
— Me, now

REST or OpenAPI

  • Front-end and back-end teams and codebases are very separate.
  • Changes happen in isolation.
  • There is code generation for type safety with OpenAPI Generator, but it often sucks.
  • Uses HTTP semantics, although POST means create and PUT means update. I think.
  • No WebSockets.

GraphQL

  • Front-end and back-end teams and codebases are very separate.
  • Changes must follow the same API contract.
  • There is code generation for type safety with GraphQL Code Generator.
  • Everything uses POST by default.
  • Supports subscriptions.

tRPC

  • Front-end and back-end is in one codebase, written in TypeScript.
  • Changes in the back-end routes affect the front-end code, invalidating type checks and requiring immediate action.
  • Queries use GET, mutations use POST.
  • Supports subscriptions.
